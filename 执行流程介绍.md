# 执行流程介绍

本文档是本仓库的从零执行说明，面向需要检查代码是否能跑通、或按论文路线复现实验的使用者。项目对应论文“弱标签、跨受试者 EEG 焦虑识别”主线：受试者级 EEG 频谱特征、联合约束神经网络、固定超参数传统机器学习基线、结构消融和外部兼容性验证。

## 0. 运行范围说明

本仓库不包含真实 EEG 原始数据、真实 `subject_features.csv`、checkpoint 或大规模实验输出。因此“能完整运行”分为两层：

- **代码完整性验证**：使用 `tests/fixtures/subject_features` 中的合成小数据，验证导入、配置读取、主模型训练、传统基线训练、summary 输出均能跑通。
- **论文级完整复现**：需要使用者按 `data/<dataset>/README.md` 准备授权数据，并生成真实的 `features/subject_features/<dataset>/subject_features.csv` 后运行完整训练。

如果只是检查项目是否整理成功，请优先执行第 2 节和第 3 节；如果要复现实验结果，再继续执行第 4 节以后。

## 1. 环境安装

```powershell
cd C:\Users\danking\Desktop\project\Graduation_thesis

```

也可以使用 conda：


说明：

- `pip install -e .` 用于让 `python -m anxiety_eeg...` 形式的模块入口稳定可用。
- 若只运行 `scripts/*.py`，脚本会自动加入 `src` 路径；但外部验证和特征提取模块仍建议先安装 editable 包。

## 2. 最小 smoke 验证

无真实数据时，用合成 fixture 验证主链路：

```powershell
python scripts/run_smoke.py --device cpu
```

预期输出位于：

```text
outputs/smoke/joint/summary_aggregate_joint_constraint.json
outputs/smoke/baselines/summary_aggregate_traditional_baselines.json
```

该命令会完成：

- 读取 `tests/fixtures/subject_features` 下 3 个合成数据集。
- 用 1 个 seed、1 个 epoch 跑通 JointConstraintNet。
- 用同一合成数据跑通 `logreg_l2` 传统基线。
- 将结果写入 `outputs/smoke/`，不会写入 `data/`。

也可以单独验证配置文件入口：

```powershell
python scripts/train_joint.py --config configs/smoke.json
```

如需单独验证传统基线：

```powershell
python scripts/train_baselines.py --features-root tests/fixtures/subject_features --output-root outputs/smoke/manual_baseline --seeds 42 --models logreg_l2 --n-jobs 1
```

## 3. 静态完整性检查

建议在提交或交付前执行：

```powershell
python -m compileall src scripts tests
rg "E:\\|D:\\|dataset_code_11" src scripts configs
python scripts/train_joint.py --help
python scripts/train_baselines.py --help
python scripts/run_ablations.py --help
python scripts/prepare_features.py
```

检查目标：

- `compileall` 通过，说明 Python 文件语法可导入编译。
- `rg` 不应返回结果，说明核心代码没有残留本机硬编码路径。
- 各入口脚本 `--help` 能正常显示，说明命令行参数和包导入可用。
- `prepare_features.py` 会打印公开数据集特征提取命令清单。

还应确认仓库中没有误放真实数据或模型权重。除 `outputs/` 外，不应提交 `.fif`、`.set`、`.edf`、`.bdf`、`.mat`、`.xlsx`、`.pt`、`.pth`、`.ckpt` 等文件。

## 4. 真实数据准备

真实复现时，先阅读对应数据目录：

- `data/original_local/README.md`：课题组/本地授权数据，不提供公开链接。
- `data/ds003478/README.md`：OpenNeuro ds003478。
- `data/ds007609/README.md`：OpenNeuro ds007609。
- `data/ds007216/README.md`：OpenNeuro ds007216，用于外部审计。
- `data/mendeley_anxiety_control/README.md`：Mendeley anxiety/control 数据。

主模型需要三个内部训练数据集的受试者级特征表：

```text
features/subject_features/original_local/subject_features.csv
features/subject_features/ds003478/subject_features.csv
features/subject_features/ds007609/subject_features.csv
```

每个 `subject_features.csv` 至少需要包含：

- `dataset`：数据集名称。
- `subject`：受试者 ID。
- `score_name`：标签来源名称。
- `anxiety`：焦虑相关连续分数。
- 默认 EEG 频谱特征列：全局频带、额叶/区域频带和比值特征。

注意：真实数据、真实特征表和包含个人信息的标签文件不进入 Git；默认应放在被 `.gitignore` 忽略的 `features/` 或本地授权目录中。

## 5. 特征提取流程

查看可用提取命令：

```powershell
python scripts/prepare_features.py
```

公开数据集示例：

```powershell
python -m anxiety_eeg.features.score_ds003478 --ds-root data/ds003478 --output-dir features/subject_features/ds003478
python -m anxiety_eeg.features.score_ds007609 --output-dir features/subject_features/ds007609
python -m anxiety_eeg.features.score_ds007216 --output-dir features/external/ds007216
```

Mendeley 示例：

```powershell
python -m anxiety_eeg.external.extract_mendeley_subject_features --workbook data/mendeley_anxiety_control/EEG_data.xlsx
```

默认输出为 `features/external/mendeley/subject_features.csv`。

本地授权数据 `original_local` 需按课题组内部数据结构生成：

```text
features/subject_features/original_local/subject_features.csv
```

如果尚未拿到本地授权数据，仍可完成 smoke 验证，但不能得到论文级完整结果。

## 6. 主模型训练

完整默认配置：

```powershell
python scripts/train_joint.py --config configs/default_joint.json
```

默认配置使用 25 个随机种子，与论文主实验报告一致。常用快速检查命令：

```powershell
python scripts/train_joint.py --features-root features/subject_features --output-root outputs/check_joint --seeds 42 --epochs 1 --min-epochs 1 --patience 1 --device cpu
```

主要输入：

- `--features-root`：包含 `original_local`、`ds003478`、`ds007609` 三个子目录的特征根目录。
- `--config`：JSON 配置文件，键名与命令行参数一致。
- `--device`：`auto`、`cpu` 或 `cuda`。
- `--seeds`：随机种子列表。

主要输出：

- 每个 seed 的训练日志和模型选择结果。
- `summary_aggregate_joint_constraint.json`：跨 seed 汇总结果。
- 输出默认位于 `outputs/joint_constraint/`，也可用 `--output-root` 改写。

论文对应关系：

- 第 4 章：EEG 频谱组织特征和受试者级建模。
- 第 5 章：dataset adapter、dataset bias、global/frontal auxiliary constraints、gray-zone weighting 等联合约束策略。

## 7. 传统基线

运行全部默认传统基线：

```powershell
python scripts/train_baselines.py --features-root features/subject_features --output-root outputs/traditional_baselines --models all
```

`--models all` 只运行默认无额外依赖的固定超参数基线：

- `logreg_l2`
- `linear_svm`
- `rbf_svm`
- `random_forest`
- `extra_trees`
- `gradient_boosting`

只跑一个快速基线：

```powershell
python scripts/train_baselines.py --features-root features/subject_features --output-root outputs/check_baseline --seeds 42 --models logreg_l2 --n-jobs 1
```

如需额外运行 XGBoost，请先安装依赖并显式指定：

```powershell
python -m pip install xgboost
python scripts/train_baselines.py --features-root features/subject_features --output-root outputs/traditional_baselines_xgboost --models xgboost
```

用途：

- 与主模型使用相同 subject-level split、gray-zone 标签和输入特征。
- 用于判断神经网络结构是否提供稳定化，而不是只报告单一模型分数。

输出：

- 每个模型和 seed 的指标表。
- `summary_aggregate_traditional_baselines.json`。
- 结果默认写入 `outputs/traditional_baselines/`。

重要说明：当前传统基线为固定超参数，不包含内层交叉验证。论文中建议写作“固定超参数传统基线”或“与主模型协议一致的传统基线”，不要写成 “train-only CV”。

## 8. 消融和分析

结构消融主入口：

```powershell
python scripts/run_ablations.py --features-root features/subject_features --output-root outputs/joint_ablation_suite --skip-external
```

其他分析模块可通过 `python -m` 调用，例如：

```powershell
python -m anxiety_eeg.analysis.run_shared_subspace_logistic --help
python -m anxiety_eeg.analysis.analyze_direction_consistency --help
python -m anxiety_eeg.analysis.compare_split_protocols --help
```

说明：

- `--skip-external` 可在只检查内部训练/消融时跳过外部数据。
- `compare_split_protocols` 涉及旧的本地授权原始数据，只作为泄露风险对照，不作为公开默认流程。

## 9. 外部验证

Mendeley partial compatibility：

```powershell
python -m anxiety_eeg.external.extract_mendeley_subject_features --workbook data/mendeley_anxiety_control/EEG_data.xlsx
python -m anxiety_eeg.external.evaluate_external_mendeley --train-output-root outputs/joint_constraint --skip-scoring
```

ds007216 domain-shift 审计：

```powershell
python -m anxiety_eeg.features.score_ds007216 --output-dir features/external/ds007216
python -m anxiety_eeg.external.evaluate_external_ds007216 --train-output-root outputs/joint_constraint --skip-scoring
python -m anxiety_eeg.external.analyze_reverse_discrimination_ds007216
```

解释边界：

- Mendeley 缺少部分训练特征，只能作为 partial-overlap compatibility test。
- ds007216 主要用于方向一致性和 domain-shift 审计，不应被写成主模型最终性能证据。

## 10. 输出目录说明

默认输出均写入 `outputs/`：

```text
outputs/smoke/                 合成数据 smoke 验证
outputs/joint_constraint/      主模型完整训练
outputs/traditional_baselines/ 传统基线
outputs/joint_ablation_suite/  消融实验
outputs/check_*                手动快速检查输出
```

`outputs/` 已在 `.gitignore` 中忽略。交付仓库时一般只保留代码、配置和说明文档，不提交训练结果、缓存、模型权重或真实数据。

## 11. 常见问题

### 找不到 `anxiety_eeg` 包

先执行：

```powershell
python -m pip install -e .
```

或临时设置：

```powershell
$env:PYTHONPATH="src"
```

### smoke 能跑，真实训练报缺少 CSV

说明真实特征表还没有准备好。请确认以下路径都存在：

```text
features/subject_features/original_local/subject_features.csv
features/subject_features/ds003478/subject_features.csv
features/subject_features/ds007609/subject_features.csv
```

### 输出指标和论文不同

先确认是否使用真实数据、完整 25 seeds、默认配置和相同特征列。合成 fixture 只用于验证代码可跑，不用于复现论文数值。

### GPU 不可用

把命令中的 `--device auto` 或 `--device cuda` 改为：

```powershell
--device cpu
```
